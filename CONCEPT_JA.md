# LightVC コンセプト

> **最終更新: 2026-06-29.** 実証済みアーキテクチャとC2研究方向を反映。

## LightVCとは

LightVCは、ニューラルコーデック（DAC）の潜在空間で完結するリアルタイム声質変換システムです。軽量アダプタがECAPA話者埋め込みを条件として、ソース話者のコーデック表現をターゲット話者へ変換します。凍結DACデコーダが波形を再構成します。

```
Mic (cpal) → DACエンコーダ → soft RVQ → B1アダプタ(ECAPA cross-attn) → DACデコーダ → Speaker
                                                                    ↑
                                                      R2 fine-tuned decoder（短窓ロバスト化）
```

**Pure Rust推論**（Candle）。デプロイ時にPython不要。CUDA環境で50ms未満のレイテンシ。

## 実証済みアーキテクチャ (v1.0)

### B1 UTTEアダプタ

現行のVCモデル: ECAPA 192次元 → 32トークン → 4ヘッドcross-attention。

```
z_q [B, 1024, T]
  → Conv1d(1024→256, k3)
  → ECAPA → Linear(192, 32×256) → 32トークン [B, 32, 256]
  → 4-head cross-attention(256)
  → GELU
  → Conv1d(256→1024, k3) [ゼロ初期化]
  → z_q + delta
```

- パラメータ数: 3.5M
- 学習データ: VCTK 10K same-textペア、adapter-only（generatorなし）
- 評価結果（200ペアブートストラップ信頼区間）:
  - **cross-text target SECS: 0.508**
  - **cross-text margin: +0.323**（target - source）
  - Oracle比: 79.2%（src_K1 oracle +0.542に対して）

### R2 Fine-Tuned デコーダ

凍結DACデコーダは短窓ストリーミング時に境界アーティファクトが発生（4f: SNR 1.6 dB）。R2はデコーダ末尾2ブロック（1.78M params）を短窓ロバスト化します。

```
block.2 (1.48M, 22kHz帯域) + block.3 (0.30M, 44kHz帯域) = 1.78M 学習対象
```

- 学習: VCTK 10K発話、固定ティーチャー（immutable copy）、5エポック
- 結果:
  - **4f SNR: +9.1 dB改善**（1.6 → 10.7 dB）
  - **4f MCD: -11.93**（32.05 → 20.11）
  - フル窓再構成: 変化なし（破滅的忘却なし）
  - B1 offline margin: 維持（+0.339）
  - Rust/Candle parity: RMSE = 7.3e-3

### 実行時制御（ノブ）

| 制御 | 範囲 | 効果 |
|------|------|------|
| **wet/dry mix** | 0.0–1.0 | VC強度（0 = バイパス、1 = 完全変換） |
| **τ (tau)** | 0.1–10.0 | soft RVQ温度（低い = 硬い量子化、高い = 滑らか） |

## 研究知見（何を学んだか）

### 実証されたこと

1. **cross-text汎化**: B1アダプタはsame-textペアで学習してもcross-textで完全汎化する（200ペア、Δmargin = +0.002）。same-text監督バイアスは存在しなかった。

2. **RVQ内に話者/content構造が存在**: depth surgery（200ペア）で話者情報の分布を実測:
   - d1-3: 話者情報の68%（「話者コア」）
   - d0: 12%（content + 話者の混在）
   - d5-8: 各<6%（微細ディテール、話者寄与ほぼなし）

3. **ECAPAが最適条件付け信号**: 事前学習済み話者埋め込みは、いかなるコーデック由来表現（生latent、depthフィルタlatent、コードブックヒストグラム、共起ルックアップテーブル）よりも優れる。

4. **デコーダfine-tuneが有効**: 末尾2ブロックのfine-tune（固定ティーチャー + 蒸留loss）により、フル窓品質を維持したまま短窓ロバスト性が劇的に向上。

### 否定されたこと（ネガティブ結果）

1. **学習なしVCは不可能**: コード選択は(content, speaker)の同時関数。統計量（ヒストグラム、共起ルックアップ）では因数分解できない。

2. **ref_latentトークンバンクは失敗**: 参照音声のコーデック表現（生・depthフィルタ）は、ECAPAの事前抽出話者ベクトルより条件付けとして劣る。

3. **per-depth離散code操作は最適化困難**: depth-aware adapter（V1: 8次元code射影、V2: 1024次元per-depth delta）はいずれもECAPA B1の性能に届かない。

4. **naive RVQトークン交換は失敗**: トークン妥当性 ≠ 残差チェーン妥当性。

5. **連続latent L1/MSE回帰は崩壊**: 全latent-domain lossがコードブック重心に収束する。

## C2研究方向: コンセプトボイスシンセシス

v1.0は機能するzero-shot VC（ソース → ターゲット話者）。次の目標は**コンセプトボイスシンセシス**: 抽象プリセットから自然な声を生成すること。

### ギャップ

「男性入力 → 官能的な女性声」には、話者変換以上のものが必要:
- **自然な女性manifold**: デコーダ/アダプタが多様な自然女性音声を見ている必要がある
- **属性制御**: "breathy"、"warm"、"intimate"が制御可能な次元である必要がある
- **ソース声質抑制**: 男性声の特徴を積極的に除去する必要がある

### C2ロードマップ

| フェーズ | 目標 | 状態 |
|---------|------|------|
| **C2-0** | 女性manifoldデータセット（Irodori-TTS 25K発話、500話者 × 5キャプション） | 生成中 |
| **C2-1** | キャプションラベルからのコンセプト埋め込み（neutral/soft/breathy/warm/low_tension） | 計画中 |
| **C2-2** | ドメインブリッジ（ソース声質リーク抑制、q0補正） | 計画中 |
| **C2-3** | コンセプトプリセットシステム（ボイスシンセノブ制御） | 計画中 |

### Irodori-TTSコーパス

`Irodori-TTS-600M-v3-VoiceDesign`（MIT、Flow Matching TTS + キャプション制御）を使用して、500話者から多様な女性音声コーパスを生成。キャプションテキストが属性ラベルとして機能:

```
neutral:     「落ち着いた自然な女性の声で、普通の速さで読み上げてください。」
soft:        「柔らかく穏やかな声で、優しく語りかけるように読み上げてください。」
breathy:     「息多めの甘い声で、囁くように親密な距離感で読み上げてください。」
warm:        「温かく包容力のある声で、安心させるようにゆっくりと読み上げてください。」
low_tension: 「リラックスして力の抜いた声で、少し低めのトーンで読み上げてください。」
```

これにより以下が同時に得られる:
- 大規模女性manifold（RVQ構造分析用）
- 属性ラベル付きペア（コンセプト埋め込み学習用）
- ペアワイズ比較素材（知覚実験用）

## 歴史的コンテキスト（教訓）

### 試行して放棄したもの

| アプローチ | 結果 | 教訓 |
|-----------|------|------|
| 連続latent flow matching (Phase B/C) | 重心へ崩壊 | latent上のL1/MSEは必ず崩壊する |
| VC teacher蒸留 (SynthVC方式) | ライセンスリスク + 品質上限 | teacher-free方針を採用 |
| naive RVQ depth交換 | デコーダ無効軌跡 | 残差チェーンを保存すべし |
| cross-textターゲットバンク検索 | content/speaker不可分 | フレームレベル検索では不十分 |
| ref_latentトークンバンク（真UTTE） | ECAPAが厳密に上位 | 事前学習話者抽出 >> 生コーデック |
| per-depthノブアダプタ (V1/V2) | B1以下でプラトー | per-depth構造は最適化に不適 |
| 学習なしコード統計 | margin -0.43〜-0.54 | コード選択は(content, speaker)同時関数 |

### 維持された原則

- **コーデック空間VC**: VCはDAC潜在空間で行う ✅
- **1ステップ変換**: ODEループなし、拡散サンプリングなし ✅
- **Pure Rust推論**: Candle、デプロイ時にPython不要 ✅
- **凍結コーデックバックボーン**: エンコーダ・デコーダは凍結（デコーダ末尾2ブロックはストリーミング用にfine-tune） ✅
- **MITライセンス**: 全コンポーネントがMIT/Apache/ISC ✅

## アーキテクチャサマリ

```
┌─ エンコーダ（凍結）────────────────────────────────────┐
│ DACエンコーダ: Conv1d(1→64→128→512→1024) + ResBlocks  │
│ 44.1kHz PCM → [B, 1024, T] latent @ 86 Hz             │
└─────────────────────────────────────────────────────────┘
         │
┌─ Soft RVQ ──────────────────────────────────────────────┐
│ softmax(τ=5.0)による順次再量子化                        │
│ q0（ソース）固定 → depth 1-8をsoft量子化               │
│ z_q = q0 + Σ soft_q_d                                  │
└─────────────────────────────────────────────────────────┘
         │                                    ┌─ ECAPA ─────┐
┌─ B1アダプタ ─────────────────────────────────│  192次元    │
│ Conv1d(1024→256) + cross-attn(32トークン)  │  話者       │
│ + GELU + Conv1d(256→1024) [ゼロ初期化]      │  埋め込み   │
│ 3.5Mパラメータ                              └────────────┘
└──────────────────────────────────────────────────────────┘
         │
┌─ デコーダ（R2 fine-tuned）──────────────────────────────┐
│ DACデコーダ: conv1 + block.0-1（凍結）                  │
│              + block.2-3（短窓用fine-tune）              │
│              + snake + conv2                             │
│ [B, 1024, T] → 44.1kHz PCM                              │
└──────────────────────────────────────────────────────────┘
```

## デプロイメント

- **プラットフォーム**: Windows（ASIO/WASAPI）、macOS（CoreAudio）、Linux（ALSA）
- **GUI**: egui/eframe（Pure Rust）
- **オーディオ**: cpalリングバッファストリーミング
- **レイテンシ**: CUDA環境で50ms未満（Balanced 4f）、CPU環境で約185ms（リアルタイム不可）
- **重みファイル**: `dac_44khz_finetuned.safetensors`（306MB）、`dac_quantizer.safetensors`（4.7MB）、`utte_adapter_b1.safetensors`（7.1MB）
